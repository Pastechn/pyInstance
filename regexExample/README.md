# Mac Address Filter

用于从指定文件中读取 Mac 地址

## 需求

指定一个文件，从该文件中读取有效的 Mac 地址。为了可读性，自动忽略以 `#` 开头的注释。

## 使用方法

1. 创建 Mac 地址列表文件
2. 编辑脚本，将 `whitelist.txt` 更改为创建的列表文件
3. 运行脚本，即可在控制台中看到列表形式的 Mac 地址

## 程序逻辑

没有逻辑，就是简单的正则表达式模块的调用

## 说明

这个脚本诞生的初衷是为了监控路由器上的联网设备，且自动排除白名单中的设备变化。

起初，我的做法是将白名单以列表的形式存储在程序内部，但是这样非常麻烦，因为列表只能随着脚本的更新而更新，于是诞生出了把名单分离的想法。

一开始，我是直接把 Mac 地址写进文件，逐行读取并去掉换行符和空行。这就要求一行内除了 Mac 地址和换行符外，不能有其他的字符。而Mac地址又过于抽象，要想分辨是哪个设备，只能靠注释。

后来想到了可以利用正则表达式。核心是 `[0-9,a-z]{2}:[0-9,a-z]{2}:[0-9,a-z]{2}:[0-9,a-z]{2}:[0-9,a-z]{2}:[0-9,a-z]{2}`，然后使用 `group(0)` 提取出有效地址。这样做的好处是支持注释，可以有整行注释，也可以有行后注释，对于后期白名单的修改提供了很大的便利。
